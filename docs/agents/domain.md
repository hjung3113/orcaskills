# Domain Docs

## Before exploring

Read the repository root `CONTEXT.md` when it exists. Also read ADRs under `docs/adr/` that apply to the area being changed. If either is absent, proceed without comment.

## Repository layout

This repository uses a single domain context. Future domain vocabulary belongs in the root `CONTEXT.md`; architecture decisions belong in `docs/adr/`.

## Vocabulary

Use terms defined in `CONTEXT.md` consistently. When work conflicts with an existing ADR, surface the conflict explicitly rather than silently overriding the decision.
